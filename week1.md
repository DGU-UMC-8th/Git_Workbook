## 로컬 저장소 vs 원격 저장소
- **로컬 저장소**: 내 노트북
- **원격 저장소**: 원격으로 접근 가능한 저장소

## Git vs GitHub
- **Git**: 우리의 로컬 시스템에 접근해서 내가 원하는 루트 디렉토리를 로컬 저장소로 만들고, 버전 관리를 도와주는 시스템
- **GitHub**: 개인 로컬 서버에 있는 Git 저장소 시스템을 공유하고, 기록하도록 도와주는 온라인 DB
  - => Git 저장소를 관리하는 클라우드 기반 호스팅 서비스

## 버전 관리란?
- 협업 시 파일 버전을 동일하게 유지하기 위함

## 자주 쓰는 Git 명령어
| 명령어 | 설명 |
|--------|------|
| `git add` | 작업 디렉토리에서 스테이징 영역으로 변경사항을 이동 |
| `git branch` | 저장소 내에 격리된 개발 환경 생성 |
| `git checkout` | 특정 커밋 혹은 브랜치로 이동 |
| `git clone` | 기존 Git 저장소의 복사본 생성 |
| `git commit` | 스테이징 된 스냅샷을 가져와 프로젝트 기록에 커밋 |
| `git fetch` | 원격 저장소의 브랜치와 파일을 최신 버전으로 업데이트 |
| `git init` | 새로운 Git 저장소를 초기화 |
| `git log` | 해당 브랜치에 대한 커밋 기록 탐색 |
| `git merge` | 분리된 브랜치의 변경 사항 통합 |
| `git pull` | 원격 저장소의 브랜치와 파일 다운로드 후 전체 브랜치에 병합 |
| `git push` | 로컬 저장소의 변경 사항을 원격 저장소에 반영 |
| `git rebase` | 브랜치를 이용하여 병합 커밋 없이 새 커밋 히스토리 생성 |
| `git remote` | Git 원격 연결 |
| `git status` | 작업 디렉토리와 스테이징된 스냅샷 상태 표시 |

## Git 버전 관리 과정
```bash
git init -> git add -> git commit -> git push
```

## 파일 상태 변화 과정
1. **Untracked/Unmodified** → 파일 수정
2. **Modified** → `git add`
3. **Staged** → `git commit`
4. **Committed** → `git push`

## GitHub 장점
- 변경 사항에 대한 버전 관리 지원
- 여러 사용자가 파일 충돌 없이 협업 가능
