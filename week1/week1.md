### 📁 1. 파일의 변경 사항을 추적

저번주 수요일에 `졸업논문_ver1.hwp` 를 작업한 후 토요일에 이어서 `졸업논문_ver2.hwp` 를 작업했다고 가정해봅시다.

우리가 Git을 사용하지 않는다면 컴퓨터에는 사진처럼 `졸업논문_ver1.hwp` 와 `졸업논문_ver2.hwp` 가 모두 저장되어 있어야 하겠죠!

그런데 만약 이렇게 계속 작업하다가 `졸업논문_ver1000.hwp` 까지 나와버린다면 … 어떨까요?

우리는 언젠가 저장한 `졸업논문_ver503.hwp` 를 찾기 위해 몇백 개의 논문 제목을 모두 흝으며 스크롤을 올려야 합니다. 또한 우리가 1000개의 문서를 모두 필요로 하지 않는 상황에서 이 수많은 문서를 전부 저장하려면 컴퓨터 저장 공간도 불필요하게 낭비되겠죠.

이 때 `Git` 을 이용하면 한 파일 내에서 일어난 변화들을 전부 `Git` 이라는 시스템을 이용해 기록할 수 있습니다.

**정확히 언제 파일을 수정했는지**,  **그 때 파일 내에서 어떠한 변화가 있었는지** 기록해주기 때문에 **특정 시점에서 이전 내용을 언제든지 다시 꺼내볼 수 있어요.** 또한 기록은 **로그 형식으로 관리되기 때문에 파일을 여러 개 저장하지 않아도 됩니다!**

이러한 **`파일 내 변화를 기록하고 찾아주는 것`** 을 **`파일의 변경 사항들을 추적`** 한다고 부르는 것입니다.

### 👥 2. 여러 사용자 간에 해당 파일들의 작업들을 조율

우리가 Git을 이용할 때는 [**Github](https://github.com)와 같은 원격 저장소를 활용**하게 됩니다. 

원격이라는 단어가 어렵게 느껴질 수 있지만 **내 컴퓨터를 로컬(local)**, 인터넷 상 저장소인 **Github를 원격(remote)**으로 이해해보면 쉽습니다.

Github가 인터넷 상에 있다고 설명드렸는데, 바로 이러한 점 덕분에 협업할 때 Git을 자유롭게 사용할 수 있습니다.

한 파일이 Github(원격)에 올라가 있다고 가정했을 때 여러 명의 사용자가 해당 파일을 각자의 컴퓨터(로컬)에 다운로드한다면 어떨까요? **한 파일을 동시에 여러 사용자의 컴퓨터에서 접근할 수 있을 겁니다.**

Github를 이용해 파일 또는 폴더를 다운받으면 **`해당 파일(또는 폴더)의 기록 또한 사용자의 컴퓨터(로컬)에서 살펴볼 수 있습니다`**. **모든 사용자가 완전히 동일한 작업 내용을 공유**할 수 있으니 **원활한 협업이 가능**해지겠죠?

**`여러 사용자 간에 해당 파일들의 작업들을 조율한다는 말`**은 곧 **`여러 사용자가 특정 내용에 대해 접근하고 공동으로 작업할 수 있는 것`**  이겠죠?

### 💻 3. 분산 버전 관리 시스템

그렇다면 `분산 버전 관리 시스템`이 뭘까요? 말이 너무 어려우니 공식 문서의 내용을 함께 봅시다.

> **Division VCS (분산형 버전 관리 시스템)**
> 
> 
> 중앙에서 관리하고 있던 모든 이력을 가진 저장소 전체를 복사하여 사용자의 컴퓨터로 가져와 사용한다.
> 

아까 Github를 이용해 파일이나 폴더를 다운받으면 **해당 파일(또는 폴더)의 기록도 함께 불러온다**고 했던 부분을 기억하시나요? 

이렇게 **`파일이 수정되고 기록하는 시점`**을 우리는 `버전`이라고 부릅니다. 그렇다면 이제 분산 버전 관리 시스템을 다시 한번 이해해볼까요?

Git을 이용하면 **여러 명이 Github(원격)에 있던 동일한 내용을 본인의 컴퓨터로 각각 불러와 개별적으로 작업**할 수 있다고 설명드렸는데요. 이 부분을 분산으로 생각하신다면 이해가 쉬울 것 같습니다. **파일이 수정되고 기록하는 시점**이 버전이라면, **작업한 내용을 기록해 버전으로서 반영**하는 것은 버전 관리라고 볼 수 있을 겁니다. 

그렇다면 `분산 버전 관리 시스템`은 곧 `여러 명이 한 파일(또는 폴더)에 접근해 개별적으로 작업하고, 작업 결과를 기록으로서 반영하는 시스템`으로 이해해볼 수 있겠습니다.
