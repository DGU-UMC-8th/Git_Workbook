### Git & GitHub  

#### **Git**  
로컬 시스템에 접근하여 로컬 저장소를 만들고 버전 관리를 도와주는 시스템  

#### **GitHub**  
개인의 로컬 서버의 Git 저장소를 관리하는 클라우드 기반 호스팅 서비스  

---

### **Git을 사용하는 이유**  
- 프로젝트를 진행하며 생기는 수정에 대해 버전 관리를 지원하여 체계적인 환경을 조성  
- 여러 사용자들과 협업하는 데 유리  

---

### **Git 저장소 종류**  
1. **로컬 저장소** (localhost)  
2. **원격 저장소**  

---

### **.gitignore**  
Git에서 추적하면 안 되는 파일 목록을 지정하는 파일로, [gitignore.io](https://www.toptal.com/developers/gitignore) 사이트에서 원하는 내용에 맞게 생성 가능  

---

### **파일 관리하기**  
#### **파일을 Git에 추가하고 업로드하기**  
```bash
git add .  # 파일을 스테이징 상태로 만듦
git commit -m "커밋 메시지"  # 변경 사항을 기록
git push origin 브랜치  # GitHub에 업로드
